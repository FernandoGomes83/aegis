### SEC-REQ-INPUT-01: Schema Validation on All Endpoints

**{{i18n.labels.derives_from}}**: `aegis/framework/security/security-requirements.yaml` — input-validation

**{{i18n.labels.acceptance_criteria}}**:

1. The system SHALL validate all API inputs against an explicit schema using a schema validation library before any business logic executes.
2. Every field SHALL have a defined type, minimum/maximum length or value, and expected format or enumeration.
3. Requests containing any field that fails validation SHALL be rejected with HTTP 400 and a response body listing the specific failing fields, without exposing stack traces or internal details.
4. The total request body size SHALL be limited at the middleware level to prevent denial-of-service via payload flooding.

### SEC-REQ-HEADERS-01: Security Headers on All Responses

**{{i18n.labels.derives_from}}**: `aegis/framework/security/security-requirements.yaml` — headers

**{{i18n.labels.acceptance_criteria}}**:

1. The system SHALL include HSTS (max-age ≥ 63,072,000s, includeSubDomains), X-Frame-Options: SAMEORIGIN, X-Content-Type-Options: nosniff, Referrer-Policy: origin-when-cross-origin, and Permissions-Policy on every response.
2. A Content-Security-Policy header SHALL be configured with `default-src 'self'` as baseline; any additional sources SHALL be documented and justified.
3. CORS headers SHALL explicitly enumerate permitted origins and SHALL NOT use the wildcard Access-Control-Allow-Origin on any endpoint that accepts authenticated requests.

### SEC-REQ-DEPS-01: Dependency Security

**{{i18n.labels.derives_from}}**: `aegis/framework/security/security-requirements.yaml` — dependencies

**{{i18n.labels.acceptance_criteria}}**:

1. The dependency lockfile SHALL be committed to version control and kept reproducible.
2. Automated vulnerability scanning SHALL run on every CI pipeline run.
3. Builds with known critical or high-severity vulnerabilities SHALL fail until the vulnerability is remediated or explicitly accepted with documented justification.
4. Critical vulnerability patches SHALL be applied within 7 days of disclosure; high-severity within 30 days.
